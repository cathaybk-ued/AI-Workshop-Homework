# AI Workshop Homework

CathayBK UED — AI 工作坊學員作業繳交區（試跑場 + 正式場）。

請將作業放入對應資料夾，並依照下方命名規則命名檔案。

---

## 一、資料夾結構

```
AI-Workshop-Homework/
├── 試跑場/      # 試跑場學員作業放這裡
└── 正式場/      # 正式場學員作業放這裡
```

---

## 二、檔案命名格式

**格式：** `場次_梯次_姓名_作業編號.副檔名`

| 欄位 | 說明 | 範例 |
|------|------|------|
| 場次 | `試跑場` 或 `正式場` | `正式場` |
| 梯次 | 場次日期或梯次代號（無則填 `T0`） | `0714` |
| 姓名 | 中文全名，不加空格 | `王小明` |
| 作業編號 | `HW1`、`HW2`…（依課程指定） | `HW1` |
| 副檔名 | 依實際檔案類型 | `.pdf` / `.zip` / `.md` |

**範例：**

```
試跑場_T0_王小明_HW1.pdf
正式場_0714_陳美惠_HW2.zip
```

規則：
- 欄位之間一律用底線 `_` 分隔，欄位內不要有空格。
- 一份作業一個檔案；多檔案請壓成一個 `.zip`。
- 檔名請勿使用特殊符號（`/ \ : * ? " < > |`）。

---

## 三、如何上傳

### 方式 A：GitHub 網頁上傳（已被加為協作者者）

1. 進入對應資料夾（`試跑場/` 或 `正式場/`）。
2. 點右上角 **Add file → Upload files**。
3. 拖曳你的作業檔案進去。
4. 下方 **Commit changes** 直接送出。

### 方式 B：Fork + Pull Request（一般學員，推薦）

1. 點右上角 **Fork**，複製一份到自己的帳號。
2. 在自己的 fork 中，進入對應資料夾上傳作業檔案（同方式 A 步驟 2–4）。
3. 回到本專案，點 **Pull requests → New pull request**，選你的 fork 分支送出。
4. 講師 review 後合併。

> 沒有 GitHub 帳號者，請先至 https://github.com 免費註冊。
